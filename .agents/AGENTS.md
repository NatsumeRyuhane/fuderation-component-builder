# Fuderation Workshop Component

This repository is a **one-component-per-repo template** for building a single
Fuderation Workshop component — an interactive UI widget that renders inside a
chat message in a storyline. Author source in `src/`; `npm run build` assembles
the importable `component.json`.

## Skills

When working in this repo, always load and follow these skills:

- **fuderation-component-builder** — the primary workflow for creating, debugging, and iterating on the component. Defines the `src/` authoring convention, the build (`npm run build` → `component.json`), the bridge function DSL reference, and the phased build process.
- **frontend-design** — design tokens, component patterns, and styling constraints. Use when writing or refining the HTML/CSS.

## Project structure

```text
src/
  markup.html      # HTML only               -> component.html
  styles.css       # styles                  -> component.css
  script.ts | .js  # TS compiled OR JS verbatim -> component.script
  ai_prompt.md     # AI supplementary prompt -> component.ai_prompt
  meta.json        # { "name", "description" }
component.json     # build output (generated, gitignored) — import into Workshop
```

## Rules

- Follow the phased build workflow from `fuderation-component-builder` — do not jump straight to a finished component.
- Author in `src/`; never hand-edit `component.json` (it is generated by `npm run build`).
- Use **either** `src/script.js` (verbatim, DSL mode) **or** `src/script.ts` (compiled by esbuild, iframe mode) — never both. Bridge functions are injected globals; never `import` them.
- When the user provides existing component code, split it into the `src/` files before making changes.
- Keep styles chat-bubble-friendly. No networking, no real auth, no real payment.
- Run `npm run build` after changes and confirm it passes validation.
- CI: pushing to `main` runs `.github/workflows/build.yml`, which rebuilds and commits `component.json` back to `main`; pull requests validate the build only. Do not hand-commit `component.json`.
